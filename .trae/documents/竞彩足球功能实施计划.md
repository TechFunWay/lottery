# 竞彩足球功能实施计划

## 一、项目现状分析

### 当前架构
- **后端**: Go + Gin + GORM + SQLite
- **前端**: Vue 3 + TypeScript + Vite + TailwindCSS + ECharts

### 当前支持的彩票类型
| 类型 | 号码结构 | 奖级 | 自动抓取 |
|------|---------|------|---------|
| 双色球 | 6红+1蓝 | 6级 | ✅ |
| 大乐透 | 5前+2后 | 7级 | ✅ |
| 福彩3D | 3位数字 | 3级 | ❌ |
| 排列3 | 3位数字 | 2级 | ❌ |
| 排列5 | 5位数字 | 1级 | ✅ |
| 七乐彩 | 7主+1特别 | 7级 | ✅ |
| 七星彩 | 6红+1蓝 | 6级 | ❌ |

### 核心数据模型
- `PurchaseRecord` - 购买记录（号码存为JSON）
- `DrawResult` - 开奖结果（号码存为JSON）
- `WinningRecord` - 中奖记录（固定奖级+奖金）

## 二、竞彩足球与现有彩票的核心差异

| 维度 | 数字彩票 | 竞彩足球 |
|------|---------|---------|
| 投注对象 | 号码组合 | 足球比赛结果 |
| 号码结构 | 固定格式数字 | 比赛选择+玩法选项 |
| 奖级 | 固定等级 | 基于赔率计算 |
| 开奖 | 统一开奖 | 比赛结束后自动出结果 |
| 串关 | 不支持 | 支持（2串1、3串1等） |
| 赔率 | 无 | SP值/固定赔率 |

### 竞彩足球支持的玩法
1. **胜平负** - 预测主胜(3)/平(1)/客胜(0)
2. **让球胜平负** - 让球数 + 胜平负
3. **比分** - 预测具体比分（1:0, 2:1, 3:3 等）
4. **总进球数** - 预测总进球（0/1/2/3/4/5/6/7+）
5. **半全场** - 预测半场+全场结果（胜胜/胜平/胜负/...共9种）

## 三、技术方案：独立数据模型

由于竞彩足球与数字彩票差异巨大，采用**独立数据模型**方案，不强行复用现有模型，但复用现有基础设施（认证、用户、统计框架）。

### 3.1 新增数据模型

#### FootballMatch（足球比赛）
```go
type FootballMatch struct {
    ID           uint      `json:"id" gorm:"primarykey"`
    CreatedAt    time.Time `json:"created_at"`
    MatchID      string    `json:"match_id" gorm:"uniqueIndex"`  // 外部比赛ID
    IssueNumber  string    `json:"issue_number"`                  // 场次号（如 周一001）
    League       string    `json:"league"`                        // 联赛名称
    HomeTeam     string    `json:"home_team"`                     // 主队
    AwayTeam     string    `json:"away_team"`                     // 客队
    MatchTime    time.Time `json:"match_time"`                    // 比赛时间
    Status       string    `json:"status"`                        // 未开赛/已完赛
    HomeScore    int       `json:"home_score"`                    // 主队进球
    AwayScore    int       `json:"away_score"`                    // 客队进球
    HalfHomeScore int      `json:"half_home_score"`               // 半场主队进球
    HalfAwayScore int      `json:"half_away_score"`               // 半场客队进球
    Handicap     float64   `json:"handicap"`                      // 让球数（让球胜平负用）
    Source       string    `json:"source"`                        // manual/auto
}
```

#### FootballBet（竞彩投注记录）
```go
type FootballBet struct {
    ID           uint      `json:"id" gorm:"primarykey"`
    CreatedAt    time.Time `json:"created_at"`
    UpdatedAt    time.Time `json:"updated_at"`
    DeletedAt    gorm.DeletedAt `json:"-" gorm:"index"`
    UserID       uint      `json:"user_id" gorm:"not null;index"` // 用户ID
    BetType      string    `json:"bet_type"`                      // 单关/2串1/3串1/...
    Amount       float64   `json:"amount"`                        // 投注金额
    Multiple     int       `json:"multiple"`                      // 倍数
    Status       string    `json:"status"`                        // 待开奖/已中奖/未中奖
    Selections   string    `json:"selections" gorm:"type:text"`   // JSON: 投注选项详情
    Remark       string    `json:"remark"`
    WinAmount    float64   `json:"win_amount"`                    // 中奖金额
}
```

**Selections JSON 结构示例：**
```json
[
  {
    "match_id": "周一001",
    "play_type": "胜平负",
    "selection": "3",
    "odds": 1.85
  },
  {
    "match_id": "周一002",
    "play_type": "让球胜平负",
    "handicap": -1,
    "selection": "3",
    "odds": 2.10
  }
]
```

### 3.2 数据库迁移

在 `migrations/upgrade.go` 中添加 `v1.1.0` 升级脚本，创建新表：
- `football_matches`
- `football_bets`

## 四、后端实现步骤

### 4.1 数据模型层 (`backend/models/`)
1. 新建 `football.go`，定义 `FootballMatch` 和 `FootballBet` 模型
2. 定义竞彩玩法常量和枚举

### 4.2 中奖规则引擎 (`backend/rules/`)
1. 新建 `football_calculator.go`
2. 实现各玩法的中奖判定逻辑：
   - `CalculateWinDrawLoss()` - 胜平负判定
   - `CalculateHandicapWinDrawLoss()` - 让球胜平负判定
   - `CalculateScore()` - 比分判定
   - `CalculateTotalGoals()` - 总进球数判定
   - `CalculateHalfFull()` - 半全场判定
3. 实现串关中奖计算：所有场次都命中才算中奖，奖金 = 投注金额 × 各场赔率乘积

### 4.3 服务层 (`backend/services/`)
1. 新建 `football_service.go`
2. 实现比赛数据服务：
   - `CreateMatch()` - 创建比赛
   - `GetMatches()` - 查询比赛列表
   - `UpdateMatchResult()` - 更新比赛结果
   - `FetchMatches()` - 从API抓取赛程
   - `FetchMatchResults()` - 从API抓取比赛结果
3. 实现投注服务：
   - `CreateBet()` - 创建投注记录
   - `GetBets()` - 查询投注列表
   - `DeleteBet()` - 删除投注
   - `CalculateBetWinning()` - 计算投注中奖
   - `RecheckAllBets()` - 重新检查所有投注

### 4.4 处理器层 (`backend/handlers/`)
1. 新建 `football_handler.go`
2. 实现API接口：
   - `POST /api/football/matches` - 创建/录入比赛
   - `GET /api/football/matches` - 获取比赛列表
   - `PUT /api/football/matches/:id` - 更新比赛结果
   - `DELETE /api/football/matches/:id` - 删除比赛
   - `GET /api/football/matches/fetch` - 自动抓取赛程
   - `POST /api/football/matches/fetch-results` - 批量抓取比赛结果
   - `POST /api/football/bets` - 创建投注
   - `GET /api/football/bets` - 获取投注列表
   - `PUT /api/football/bets/:id` - 更新投注
   - `DELETE /api/football/bets/:id` - 删除投注
   - `POST /api/football/bets/recheck` - 重新检查中奖

### 4.5 路由注册 (`backend/main.go`)
在 `authorized` 路由组下添加：
```go
football := authorized.Group("/football")
{
    football.POST("/matches", handlers.CreateFootballMatch)
    football.GET("/matches", handlers.GetFootballMatches)
    football.PUT("/matches/:id", handlers.UpdateFootballMatch)
    football.DELETE("/matches/:id", handlers.DeleteFootballMatch)
    football.GET("/matches/fetch", handlers.FetchFootballMatches)
    football.POST("/matches/fetch-results", handlers.FetchFootballResults)
    football.POST("/bets", handlers.CreateFootballBet)
    football.GET("/bets", handlers.GetFootballBets)
    football.PUT("/bets/:id", handlers.UpdateFootballBet)
    football.DELETE("/bets/:id", handlers.DeleteFootballBet)
    football.POST("/bets/recheck", handlers.RecheckFootballBets)
}
```

### 4.6 数据库升级 (`backend/migrations/upgrade.go`)
添加 `v1.1.0` 升级脚本，自动创建新表。
