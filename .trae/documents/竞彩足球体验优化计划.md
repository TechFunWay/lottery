# 竞彩足球体验优化计划

## 问题分析

### 1. 统计分析不包含竞彩足球
当前 `StatisticsView.vue` 的筛选器只有数字彩票类型（双色球、大乐透等），后端统计API（`statsApi.overview/prizes/trends`）基于 `PurchaseRecord`/`DrawResult`/`WinningRecord` 模型，完全不包含竞彩足球数据。由于竞彩足球的数据模型独立，短期内无法整合到现有统计中。

**方案**：将"竞彩足球"菜单移到导航栏最后面（统计分析之后），避免用户误以为统计分析包含竞彩足球数据。

### 2. 竞彩足球主页缺少录入入口
当前 `FootballView.vue` 的问题：
- 桌面端只有小字"查看全部 →"链接，不够醒目
- 移动端底部有"比赛管理"和"投注记录"按钮，但桌面端没有对应入口
- 新用户进入竞彩足球页面后，不知道从哪里录入比赛数据和投注记录

**方案**：在竞彩足球主页增加醒目的操作入口卡片区域。

---

## 实施步骤

### 步骤1：调整导航栏顺序
**文件**：`frontend/src/components/NavBar.vue`

将 navItems 中"竞彩足球"移到"统计分析"之后：
```typescript
const navItems = [
  { name: '仪表盘', path: '/', icon: Home },
  { name: '购买记录', path: '/purchase', icon: ShoppingCart },
  { name: '开奖管理', path: '/draw', icon: Award },
  { name: '中奖记录', path: '/winnings', icon: Gift },
  { name: '历史命中', path: '/history-hit', icon: Target },
  { name: '统计分析', path: '/statistics', icon: BarChart3 },
  { name: '竞彩足球', path: '/football', icon: Zap },
]
```

### 步骤2：优化竞彩足球主页，增加醒目的操作入口
**文件**：`frontend/src/views/FootballView.vue`

在统计卡片和近期数据之间，增加一个"快捷操作"区域，包含：
- **录入比赛** 按钮 → 跳转到 `/football/matches`（带新增模式）
- **录入投注** 按钮 → 跳转到 `/football/bets`（带新增模式）
- **抓取赛程** 按钮 → 直接调用抓取API
- **获取结果** 按钮 → 直接调用获取结果API

设计为桌面端和移动端都可见的卡片式入口，使用大图标+文字，视觉上醒目。

同时优化"近期比赛"和"近期投注"区域：
- 将"查看全部 →"改为更明显的按钮样式
- 空数据时显示引导性提示（如"点击上方按钮录入比赛数据"）

### 步骤3：移除移动端底部固定栏
由于主页已增加醒目入口，移动端底部固定栏不再必要，移除以避免重复。
